   - Initialisation du client Supabase cote navigateur (createBrowserClient de @supabase/ssr)

   - Initialisation du client Supabase cote serveur (createServerClient de @supabase/ssr)

5. src/utils/supabase/auth.ts
   - signUpUser(data) : Inscription email/mot de passe avec metadonnees (nom, role, specialite, licence)
   - logInUser(email, password) : Connexion email/mot de passe
   - logInWithGoogle() : Demarrage du flux OAuth Google, redirection vers /auth/callback
   - logOutUser() : Deconnexion
   - getCurrentUser() : Recuperer l'utilisateur courant (utile apres OAuth)
   - updateUserProfile(data) : Mettre a jour les metadonnees + mot de passe (pour completion profil Google)

[BACKEND / ROUTE API NEXT.JS]
--------------------------------------------------
6. src/app/auth/callback/route.ts
   - Route API (GET) qui intercepte le retour de Google OAuth
   - Echange le `code` contre une session Supabase (exchangeCodeForSession)
   - Recupere le type de compte (doctor / patient) depuis user_metadata
   - Redirige vers /dashboardoctlarabi ou /dashboardpatientlarabi
   - Si profil incomplet (pas de account_type) -> redirige vers /signup?step=complete

7. src/app/auth/auth-error/page.tsx
   - Page d'erreur simple si le flux OAuth echoue

=== FLUX D'AUTHENTIFICATION IMPLEMENTS ===
--------------------------------------------------

[Inscription Email/Mot de passe]
  /signup -> Formulaire -> signUpUser() -> Supabase Auth -> Redirection dashboard

[Connexion Email/Mot de passe]
  /login -> Formulaire -> logInUser() -> Supabase Auth -> Redirection dashboard

[Connexion Google (nouveau compte)]
  /login ou /signup -> Bouton Google -> logInWithGoogle() -> Google -> /auth/callback -> profil incomplet -> /signup?step=complete -> updateUserProfile() -> Redirection dashboard

[Connexion Google (compte existant)]
  /login ou /signup -> Bouton Google -> logInWithGoogle() -> Google -> /auth/callback -> profil complet -> Redirection dashboard directe
